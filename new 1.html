Key Features

Single Sign-On

Easy and seamless access to all resources.
Wordpress single sign on via any existing
saml 2.0 identity provider

Protect Your Complete Site

You can restrict your site to only logged in users by redirecting the users to your idP if logged in session is not found

Login Page

Allow Auto-Redirect to IDP from login page even user using multiple service provider

Attribute Mapping

Helps you to get user attribute from your IdP nad map them to Wordpress User attributes with support for custom attributes.

Role Mapping

Helps you to assign specific roles to users of a certain group in your IdP

Multisite Support

Consisting Multisites to share single installation. Configure IDP for all your networks

Widget/Shortcode to add IDP Login

Add a link or button anywhere on your WordPress site to allow user to authenticate via their Identity Provider

Single Logout

Allow user to logout once from any service provider even user handling multiple service provider

Multiple IDP's Supported

Configuration of multiple IDP's to authenticate the different group of users with different IDP's

Auto-Sync IdP configuration from metadata

Keep your IDP SAML Configuration and Certificates updated and in sync

Multinetwork Support

Allows multiple subdomains/subdirections to share single installation. Configure the IDP for all your sites in a network

Federation Support

Allow users to Single On into the site via their institution that is affiliated with InCommon Federation



If You want Wordpress to act as a SAML Identity Provider and Single Sign on into various SAML supported Service Providers to securely authenticate the user using the Wordpress site then you need to use WordPress as IDP Plugin.
